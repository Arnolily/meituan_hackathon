# Design Reference: Apple Maps (apple.com.cn/maps)

> 本文档提取 Apple Maps 中国官网的设计语言，用于指导后续页面的视觉与交互实现。

---

## 1. 设计哲学

- **内容优先**：页面 90% 以上区域被产品截图、设备 Mockup 或实景图占据，文字仅作点睛说明。
- **极简留白**：区块之间使用极大的垂直间距（通常 120px–200px），让每一屏只聚焦一个功能点。
- **去装饰化**：无多余边框、无渐变背景（除图片本身）、无复杂纹理。所有视觉重量来自内容本身。
- **系统一致性**：遵循 Apple 原生设计语言——圆角、毛玻璃、细腻阴影、物理动效。

## 2. 色彩系统

### 2.1 主色板

| Token | 色值 | 用途 |
|-------|------|------|
| `--bg-primary` | `#ffffff` | 页面主背景 |
| `--bg-secondary` | `#f5f5f7` | 次级区块背景（浅灰，几乎不可察觉） |
| `--bg-dark` | `#000000` | 深色模式/沉浸区块背景 |
| `--text-primary` | `#1d1d1f` | 主标题、正文 |
| `--text-secondary` | `#86868b` | 副标题、说明文字 |
| `--accent` | `#0071e3` | 链接、按钮、强调标签（Apple 蓝） |
| `--accent-hover` | `#0077ed` | 悬停状态 |
| `--surface` | `#ffffff` | 卡片、浮层面板 |
| `--border` | `rgba(0, 0, 0, 0.08)` | 极淡分割线 |

### 2.2 深色模式映射

| Token | 色值 |
|-------|------|
| `--bg-primary-dark` | `#000000` |
| `--text-primary-dark` | `#f5f5f7` |
| `--surface-dark` | `#1c1c1e` |
| `--border-dark` | `rgba(255, 255, 255, 0.1)` |

> 规则：深色模式不是简单反色，而是降低整体亮度，保持图片/地图的鲜艳度，文字使用略灰的白（`#f5f5f7` 而非纯白）以减少刺眼感。

## 3. 字体系统

### 3.1 字体栈

```css
font-family: -apple-system, BlinkMacSystemFont, "SF Pro Text", "SF Pro Display", "PingFang SC", "Helvetica Neue", Helvetica, Arial, sans-serif;
```

### 3.2 字号层级

| 层级 | 桌面端字号 | 字重 | 行高 | 用途 |
|------|-----------|------|------|------|
| Hero Title | 56px–80px | 600 (Semibold) | 1.1 | 首屏大标题 |
| Section Title | 40px–48px | 600 | 1.15 | 功能区块标题 |
| Headline | 28px–32px | 600 | 1.2 | 子模块标题 |
| Body | 17px–19px | 400 | 1.5 | 正文描述 |
| Caption | 14px–16px | 400 | 1.4 | 补充说明、标签 |
| Nav Link | 12px | 400 | 1 | 导航栏文字 |

### 3.3 排版规则

- **标题断行策略**：大标题允许在语义合理处断行，形成阶梯感；使用 `<br>` 精确控制而非依赖容器宽度。
- **颜色对比**：标题使用 `--text-primary`，副标题/描述使用 `--text-secondary`，形成自然的层级衰减。
- **无下划线链接**：正文中链接使用 `--accent` 色，无下划线，hover 时添加细微下划线或颜色加深。

## 4. 导航栏设计

### 4.1 全局导航（Global Nav）

- **高度**：44px（桌面端），紧凑不抢视觉焦点。
- **背景**：`--bg-primary` 带 `backdrop-filter: blur(20px)` 毛玻璃效果，滚动后底部出现 0.5px 的 `--border` 分割线。
- **布局**：Logo（左侧）+ 文字链接（居中/左对齐）+ 操作按钮（右侧）。
- **链接样式**：
  - 字号 12px，字重 400，颜色 `--text-primary`。
  - 无下划线，无背景色块。
  - Hover：颜色保持，无视觉变化（Apple 风格极度克制）；或仅透明度变为 0.8。
- **下拉菜单**：圆角 18px，纯白背景，带大阴影 `0 12px 40px rgba(0,0,0,0.15)`，内部链接 14px，行高 44px。

### 4.2 本地导航 / 标签栏

- 若页面需要二级导航，采用居中排列的文字标签。
- 当前激活项：无下划线，而是使用 `--accent` 色或加粗（600）表示。
- 标签间距：32px–48px，不拥挤。

## 5. 布局与间距

### 5.1 容器

- **最大宽度**：无硬性 max-width，图片和视频通常全宽出血。
- **内容安全区**：文字内容限制在约 980px–1200px 内居中，保证可读性。
- **侧边距**：桌面端 48px–80px，平板 32px，移动端 20px。

### 5.2 区块间距

- 标准 Section Padding：`120px 0`（桌面），`80px 0`（平板），`60px 0`（移动端）。
- 图文交替区块：文字与图片之间保持 80px–120px 的呼吸空间。
- 连续图片区块：间距可压缩至 40px–60px，形成画廊感。

### 5.3 对齐方式

- 文字区块通常**居中**或**左对齐**（若图片在右侧）。
- 避免两端对齐，保持自然的 rag right。

## 6. 卡片与窗口设计（核心）

### 6.0 Apple Store 风格卡牌货架（Intent 首页）

基于 Apple Store 在线商店页面提取的卡牌语言，用于需求输入页、功能入口页和演示模式首页：

- **页面底色**：使用 `#f5f5f7` 作为整页背景，卡片使用纯白 `#ffffff`，不要在首页卡片上使用强毛玻璃。
- **标题结构**：首屏采用 Apple Store 式大标题，第一行可用灰色强调上下文，第二行使用深色表达核心动作，例如“路线都称心，体验更如意”。
- **卡牌布局**：桌面端采用一张主操作大卡 + 3 张横向功能卡；移动端功能卡改为横向滚动货架。
- **卡牌外形**：圆角可放大到 `28px`，比地图浮窗更柔和；主卡最小高度可高于普通卡，形成 Apple Store 商品卡的陈列感。
- **阴影**：首页卡牌使用轻而扩散的阴影，例如 `0 18px 52px rgba(0,0,0,0.08)`，避免厚重边框和玻璃描边。
- **文案密度**：每张功能卡只承载一个卖点，标题 1 行，描述 1-2 行；不要在卡片内堆叠说明文本。
- **强调色**：小型 eyebrow 可使用 Apple Store 常见的暖色提示 `#bf4800`，主要操作仍使用 `--accent` 蓝色。
- **交互**：卡片 hover 只做轻微上浮或阴影变化，不使用旋转、炫光、复杂渐变。

### 6.1 地点卡 / 信息面板（Place Card）

这是 Apple Maps 页面最具代表性的"窗口"元素：

- **外形**：大圆角 `border-radius: 20px`（移动端 16px）。
- **背景**：`--surface` 纯白，无渐变。
- **阴影**：多层柔和阴影，模拟物理层级：
  ```css
  box-shadow:
    0 4px 16px rgba(0, 0, 0, 0.04),
    0 12px 32px rgba(0, 0, 0, 0.08),
    0 24px 64px rgba(0, 0, 0, 0.04);
  ```
- **边框**：无实体边框，仅靠阴影区分层级；或在深色模式下使用 1px `--border-dark`。
- **内部结构**：
  - 顶部：标题 + 关闭按钮（右上角圆形半透明按钮）。
  - 中部：横向滚动图片集（圆角 12px，图片间距 8px）。
  - 底部：操作按钮行（蓝色大按钮 + 更多选项），信息列表（图标 + 文字）。
- **按钮**：主按钮为填充色 `--accent`，圆角 12px–16px，文字白色，字重 600。

### 6.2 筛选标签（Pills / Chips）

- **形状**：圆角胶囊 `border-radius: 100px`。
- **默认状态**：白色背景，1px `--border` 边框，`--text-primary` 文字。
- **激活状态**：`--accent` 背景，白色文字，无边框。
- **间距**：标签之间 8px–12px，行间距 12px。
- **字号**：14px–16px，字重 400。

### 6.3 设备窗口（Mockup 容器）

- 页面大量使用 iPhone / Apple Watch / MacBook 的实物图作为 UI 的"窗口"。
- 若用 CSS 模拟设备框：
  - iPhone：圆角 40px–50px，细边框 `rgba(0,0,0,0.1)`，内部截图圆角略小。
  - 截图内部：保持真实 UI 的圆角规范（按钮 10px，卡片 16px）。

## 7. 组件规范

### 7.1 按钮

| 类型 | 样式 |
|------|------|
| Primary | `background: --accent; color: white; border-radius: 12px; padding: 12px 24px; font-weight: 600;` |
| Secondary | `background: transparent; color: --accent;` |
| Text Link | `color: --accent;` 无下划线，hover 可出下划线 |

### 7.2 列表 / 信息行

- 图标（24px–28px，单色或品牌色）+ 文字左对齐。
- 行高 48px–56px，底部 1px `--border` 分割线。
- Hover 背景：`rgba(0,0,0,0.02)`，无动画延迟。

### 7.3 图片展示

- **圆角**：截图类 12px–16px，实景类 0px（全出血）或 20px（卡片内）。
- **阴影**：仅在需要层级时加阴影，实景图通常不加。
- **比例**：保持设备原始比例，不拉伸；多设备组合时底部对齐。

## 8. 动效与交互

### 8.1 滚动行为

- **平滑滚动**：`scroll-behavior: smooth`。
- **视差/缩放**：大图在进入视口时，从 `scale(1.05)` 缓慢恢复至 `scale(1)`，营造电影感。
- **文字渐入**：标题和正文从下方 20px 处，伴随 `opacity: 0 → 1` 渐入，持续时间 600ms–800ms，缓动 `cubic-bezier(0.4, 0, 0.2, 1)`。

### 8.2 悬停反馈

- 导航链接：无反馈或仅透明度变化。
- 卡片：整体轻微上浮 `translateY(-4px)`，阴影加深。
- 按钮：背景色加深或亮度微调，过渡 200ms。
- 图片：禁止无意义的悬停放大，保持稳重感。

### 8.3 页面转场

- 单页内锚点跳转：平滑滚动。
- 若为多页：无特殊转场，保持原生浏览体验。

## 9. 响应式策略

### 9.1 断点

| 断点 | 目标设备 |
|------|----------|
| 1440px+ | 大屏桌面 |
| 1024px | 标准桌面 / 平板横屏 |
| 768px | 平板竖屏 |
| 480px | 移动端 |

### 9.2 适配规则

- **导航栏**：移动端折叠为汉堡菜单或精简为核心链接。
- **图文布局**：桌面端左右分栏（50/50 或 40/60），移动端垂直堆叠，图片优先展示。
- **字号缩放**：Hero Title 从 80px 降至 40px（移动端），保持字重不变。
- **卡片**：宽度从固定值变为 `calc(100% - 40px)`，保持 20px 边距。
- **间距**：Section Padding 等比缩减（桌面 120px → 移动端 60px）。

## 10. 关键设计原则（Checklist）

- [ ] 每一屏只讲一件事，文字不超过 3 行。
- [ ] 图片/视频占主导，文字仅占 20% 以下面积。
- [ ] 圆角统一：大容器 20px，中容器 16px，小元素 12px，胶囊 100px。
- [ ] 阴影柔和多层，避免单一层重的阴影。
- [ ] 字体只用系统字体栈，不引入第三方字体文件。
- [ ] 无装饰性元素（无图标背景、无渐变装饰块、无花哨分割线）。
- [ ] 深色模式不是简单反色，而是重新映射层级色。
- [ ] 交互反馈克制：不炫技，不干扰内容阅读。

## 12. Intent 首页表单与 Demo Mode 补充规范（2026-05-21）

本节用于约束需求输入页的实际交互质量，避免只保留 Apple 风格外观而缺少真实产品逻辑。

- **分段控件必须填满轨道**：出发类型、起点、预算、节奏这类 segmented control 使用等宽单元格，灰色轨道只作为控件容器，不允许出现按钮按文字宽度渲染后留下的大块空白。
- **最小触达尺寸**：分段控件按钮、主次 CTA、确认操作按钮最小高度统一为 `44px`，移动端不低于该值。
- **功能卡不靠空白撑高级感**：需求主卡可以有留白，但不能通过固定超大 `min-height` 制造无意义空洞；留白应服务于阅读层级，而不是让控件漂在卡片中间。
- **Demo Mode 是单屏唯一主动作**：如果页面目标是演示闭环，则“启动演示流程”是主 CTA，“分析需求”降级为次操作，避免两个并列主动作争夺注意力。
- **确认卡必须贴近触发动作**：用户完成输入或启动演示后，意图确认卡应立即出现在主表单卡之后，再展示偏好说明卡，保证“输入 → 确认 → 进入地图”的空间连续性。
- **货架卡只做辅助说明**：Apple Store 风格卡牌用于解释偏好方向、路线风格和价值点，不应压过任务入口本身；功能卡文案保持 1 个卖点 + 1 到 2 行说明。
- **移动端优先保证操作完整性**：表单区单列排布，CTA 可上下堆叠，页面本身不出现横向滚动；若使用横向卡牌货架，仅允许辅助卡片横滑，不得影响主流程按钮可见性。
- **起点位置预览必须是真地图**：首页左侧位置卡使用高德缩略地图展示当前 GPS / 上次位置 / 默认区域，不再使用纯 CSS 装饰假地图；定位成功后同步写入全局地图中心，进入地图页时形成“缩略地图展开为全屏地图”的空间连续性。

## 13. P2 智能路线与长期偏好规范（2026-05-21）

本节用于约束接入模型、个人偏好和历史复用后的高级体验，保证功能增强不破坏 Apple Maps 式清爽布局。

- **模型微调只做轻量入口**：地图路线面板内允许一行自然语言输入，例如“少走路一点 / 换成少排队 / 加一个商场”。入口应靠近当前路线摘要，不做完整聊天窗口，避免抢占地图主任务。
- **模型失败必须有本地兜底**：大模型解析失败时，前端使用关键词规则完成确定性微调，并通过 toast 说明“已用本地规则完成微调”，保证演示流程不断裂。
- **个人偏好要参与排序而非只展示**：长期偏好设置保存后，应立即影响 3 条路线的排序、匹配分和标签；路线卡展示“偏好匹配分”与 2-4 个短标签即可，不堆叠复杂解释。
- **历史路线是主动保存**：路线不会自动污染历史，只有用户点击“保存路线”后才写入本地历史；历史面板提供路线名称、时间、地点数、距离、排队和匹配分，并支持一键复用。
- **真实感距离计算保持确定性**：P2 前端使用坐标 Haversine + 步行绕行系数估算距离和步行时间，保证编辑、排序、替换后数值真实变化；不使用随机数。
- **P2 仍保持轻量地图优先**：自然语言微调、历史复用、偏好匹配都是路线浮窗的辅助能力，不新增大型侧栏聊天，也不让解释文本遮挡地图。
